# VibeESP32 - 파일 탐색기 안정화, 라이브러리 경로 최적화 및 로컬 히스토리 헬퍼 보고서

ESP32 보드의 하위 디렉토리를 포함한 전체 파일 시스템의 안정적인 조회, 라이브러리 프리셋의 표준 경로 배치, 그리고 로그인 여부와 무관하게 동작하는 로컬 저장소 기반의 히스토리 삭제/조회 기능의 구현 작업을 완료했습니다.

---

## 🛠️ 추가 및 개선 사항 요약

### 1. ⚙️ 프리셋 라이브러리 표준 설치 경로 도입 (`lib/` 폴더 강제)
- **현상**: 기존에는 `tcs34725.py` 및 `ssd1306.py` 프리셋 설치 시 보드의 루트 디렉토리에 설치되어 MicroPython 모듈 검색 경로(`sys.path`) 상에서 제대로 참조되지 않거나, 불필요하게 꼬여서 작동하지 않는 경우가 있었습니다.
- **해결**:
  - 프리셋 설치 타겟 경로를 `lib/tcs34725.py` 및 `lib/ssd1306.py`로 강제 지정했습니다.
  - 설치 전 보드 내에 잔존해 있는 구버전이나 중복된 찌꺼기 파일들(`tcs34725.py`, `/lib/tcs34725.py` 등)을 사전에 완벽히 지우고 깔끔하게 재설치합니다.
  - 이로써 MicroPython 환경에서 `from tcs34725 import TCS34725` 코드가 별도 설정 없이 바로 100% 동작합니다.

### 2. 📁 보드 파일 탐색기(VFS) 새로고침 및 삭제 동작 완전 안정화
- **현상**: 파일 삭제나 업로드 후 파일 목록이 즉시 업데이트되지 않고, 이전 파일 목록이 캐싱되어 남아있어 삭제/추가 완료 여부를 확인하기 어려웠습니다.
- **원인**: Raw REPL에서 재귀적으로 파일 목록을 구하는 `print('FILES:', _list_all())` 과정 중 ESP32 출력 버퍼의 처리 한계 시간을 넘겨 출력 도중에 REPL을 탈출(`Ctrl+B`)해 버려 목록 파싱이 유실되었던 것이 원인이었습니다.
- **해결**:
  - 업로드(`uploadFileToBoard`), 폴더 업로드(`handleFolderUpload`), 파일 삭제(`deleteBoardFile`) 시 내부에 포함되어 있던 임시 파일 리스트 쿼리문을 완전 분리하여 간결화했습니다.
  - 작업이 모두 성공적으로 끝난 후(Raw REPL 탈출 완료 시점) 넉넉한 딜레이를 두고 `refreshBoardFiles`를 안전하게 단독 재호출하도록 비동기 시퀀스를 개편했습니다.
  - 시리얼 통신을 통해 수신된 데이터 중 `FILES:` 리스트를 파싱할 때, 개행 문자(`\n`) 분리 대신 **정규식 괄호 매칭(Bracket Matching, `FILES:\s*(\[.*?\])`)** 기법을 적용하여 쪼개진 바이트가 조각나 수신되더라도 온전한 JSON 리스트가 완성될 때까지 안전하게 기다린 후 확실히 리스트를 업데이트하고 버퍼를 제거(finally)하도록 구현했습니다.
  - 보드 연결 시 최초 새로고침 지연 시간을 600ms에서 **1200ms**로 늘려 부팅 중인 보드와의 연결 초기에 조회가 유실되는 오작동을 근절했습니다.

### 3. 📜 로그인 여부와 무관한 히스토리 모드 (Local Storage Fallback)
- **현상**: Google 로그인을 해야만 히스토리가 기록되고, 로그인하지 않으면 히스토리를 전혀 확인하거나 삭제할 수 없었습니다. 또한 `.env.local` 파일에서 등호(`=`) 옆의 불필요한 공백과 큰따옴표로 인해 Firebase API Key 정보가 잘못 들어가 인증 초기화 실패 문제가 동반되었습니다.
- **해결**:
  - `.env.local` 파일의 Firebase 접속 토큰 문자열들의 공백 및 따옴표를 완전히 제거해 정상 바인딩시켰습니다.
  - 비로그인(게스트) 상태에서도 브라우저의 `localStorage`를 활용하여 최근 50개의 코드 생성/직접 실행 히스토리를 실시간 저장하고 개별 삭제 및 전체 삭제를 하실 수 있게 로직을 확장했습니다.
  - 비로그인 유저가 히스토리 탭을 선택하면, **"임시 로컬 히스토리 모드"**를 알리는 안내 배너와 로그인 유도 버튼이 상단에 노출되고, 로컬에 저장되어 있던 모든 히스토리 카드가 올바르게 렌더링되게 하여 뛰어난 사용자 경험을 제공합니다.

### 4. 🔮 AI 컬러 센서 코드 생성 시 Standalone 드라이버 강제
- **해결**: AI 백엔드(`api/chat.ts`)의 시스템 프롬프트를 고도화하여, 사용자가 AI에게 TCS34725 컬러 센서 코드를 작성해 달라고 요청할 때 외부 `tcs34725.py`에 대한 의존성(`import`)을 띄우지 않고, `class TCS34725` 드라이버를 코드 상단에 인라인으로 직접 선언하여 단독 실행 가능한 완결형 코드를 출력하도록 수정했습니다.

### 5. 🌦️ "서울 날씨" 및 "TCS34725" 추천 템플릿 코드 고정
- **현상**: 추천 템플릿을 클릭하면 예시코드가 편집기 창에 정상적으로 올라왔으나, 자연어 입력창에 해당 프롬프트 문구가 채워진 상태로 "생성" 버튼을 다시 누르면 `neopixel` 등 하위 포괄 키워드 매칭에 걸려 엉뚱하게 무지개 효과(네오픽셀) 코드로 덮어씌워지는 문제가 있었습니다.
- **해결**:
  - `api/chat.ts` 백엔드 서버 라우터에 `TCS34725_CODE` 와 `WEATHER_CODE` 정적 반환 로직을 추가했습니다.
  - 이 두 가지 복합 센싱 기능이 포함된 구체적 키워드가 입력되는 경우, 범용 키워드(`neopixel`, `wifi`) 조건문보다 **가장 최우선적으로 인터셉트**되도록 라우터 분기 우선순위를 맨 위로 끌어올렸습니다.
  - 이로써 추천 템플릿 카드를 선택한 직후 사용자가 고의 또는 실수로 "생성" 단추를 다시 클릭하더라도, 에디터에 로드된 원본 예제 템플릿 코드가 유실되지 않고 안전하게 그대로 고정되어 출력됩니다.
